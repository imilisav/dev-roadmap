Now that you've gotten familiar with some basic concepts, tools and skills, now we'll get into the juicy details. In web development, people like to classify various skills or tools into two sides of a website, a client side and a server side. It is very likely you'll encounter working with concepts from both sides. I highly encourage going through both sides of the roadmap. Although, feel free to skip ahead to whichever one you want.

Client-Side Server-Side

Client-Side üëÄ

Client-side refers to the part of the website, which you can see and interact with.

HTML

Hyper Text Markup Language or known as HTML, is used to layout the text and other related components such as images, sound and more. onto the web page. HTML elements can have attributes which can be modified by CSS, which is the next topic to be talked about.

Learn HTML

CSS

CSS is used to style the HTML and lay it out. For example, you can use CSS to alter the font, color, size, and spacing of your content, split it into multiple columns, or add animations and other decorative features.

Learn CSS

Pre-Processors üíø

A Pre-Processor is code that is more developer friendly to write, but will be compiled into CSS for a website.

A popular pre-processor for CSS is SASS.

Frameworks üñºÔ∏è

Frameworks make the dream work! Haha, on a serious note, frameworks are used to create a default style for developers so they can focus on the content creation rather than handling the styling for multiple browsers.

A popular framework for CSS is Bootstrap.

Architecture üìê

In simple terms, architecture refers to a industry standard of writing CSS code. A common architecture is the Block Element Modifier.

JavaScript

JavaScript is used to make the content more alive, such as moving items around or updating content on the website based on user interaction.

Learn JavaScript

Frameworks üñºÔ∏è

JavaScript frameworks are code that has been written for the common developer to do routine tasks correctly and to standard.

Learn React
Learn Angular

Server-Side refers to the software in the website that interacts with databases, is able to run on a server, handles user requests and redirects the user to different pages on the website.

Pick Your Poison Language of Choice ü§î

This is almost identical to the programming language you chose to learn about data structures & algorithms, except choose a language that can be used for server side logic. Learn its syntax, and practicing your data structures & algorithms with this language.

Package Managers üì¶

This is identical to the package manager used in the client-side except your language of choice has its own package manager. Read the tutorials regarding your language's package manager. For example for Python, pip is used.

Standards, and More Standards üìù

Just like with CSS architecture, your language of choice has a strict way to be written. At the time of writing, for python, the style guide PEP 8 is used.

Databases üóÑÔ∏è

This is where the data of the website lives. Databases are storage systems that act faster than hard disk storage, which is crucial for telecommunications.

The information can be put in two formats: SQL, noSQL

Relational/SQL

SQL ‚Äì Structured Query Language, is the language used to pull data from relational databases, which is in other words, a huge table of data. Figure out what the language you chose requires to interface with the SQL database

Learn PostgreSql, an industry favourite

NoSQL

NoSQL ‚Äì Not only Structured Query Language, is a different way of storing data in a text format, such as JavaScript's Object Notation (JSON). Figure out what the language you chose requires to interface with a NoSQL database.

Learn mongoDB, an industry favourite

RESTful APIs

Representational State Transfer (REST) is a software architectural style that defines a set of protocols that a developer has to follow by for maximum security. Figure how your language uses this protocol.

REST in more detail
